# 权益类基金持仓穿透分析模块设计方案

## 一、模块定位

**目标**：替代原有的"第五部分 持仓穿透分析"，提供更深入、更可视化的持仓分析

**核心价值**：
- 识别基金经理的投资框架稳定性
- 评估交易能力（逃顶/抄底能力）
- 风险预警（高估/行业集中度/压力测试）

---

## 二、功能模块设计

### 模块2：前十大持仓股历史变化可视化
**目的**：了解基金经理的投资框架是否稳定，是否存在频繁交易

**数据来源**：
- 接口：`ak.fund_portfolio_hold_em(symbol, date)`
- 时间范围：根据用户选择，但**最多显示最近5年（20个季度）**
  - 如果用户选择"成立以来"且基金成立>5年，只显示最近5年
  - 如果用户选择"现任经理"且经理任期>5年，只显示最近5年
  - 如果基金成立<5年或经理任期<5年，按实际情况显示
- 数据粒度：按季度

**可视化方案**：
```python
# 方案A：动态热力图（推荐）
# X轴：季度时间线
# Y轴：前十大持仓股（按最新持仓排序）
# 颜色：占净值比例（热力图）
# 标记：新进🟢、退出🔴、增持⬆️、减持⬇️

# 方案B：桑基图（Sankey Diagram）
# 展示持仓流动方向：上期持仓 → 本期持仓
# 流线粗细：持仓占比

# 方案C：河流图（Stream Graph）
# 展示每只重仓股的占比变化曲线
```

**关键指标**：
- **持仓留存率**：连续N个季度保留在持仓中的股票数（最多20个季度）
- **平均持仓周期**：每只股票的平均持有季度数
- **换手率**：新进+退出股票 / 总持仓数
- **风格标签**：
  - `持股周期 ≥ 6季度` → "长期价值投资型"
  - `2季度 ≤ 持股周期 < 6季度` → "中期平衡型"
  - `持股周期 < 2季度` → "高频交易型"

**数据范围控制**：
- 基于最近5年（20个季度）的持仓数据
- 如果基金成立<5年，使用全部可用数据

### 模块3：交易能力评估（新买/卖的30天走势）
**目的**：判断基金经理是否有逃顶/抄底能力，还是接盘侠

**数据来源**：
- 持仓变动：`ak.fund_portfolio_hold_em()` 对比连续两期持仓
- 股票走势：`ak.stock_zh_a_hist()` 获取历史收盘价
- 评估窗口：交易后30个交易日（约1.5个月）

**实现逻辑**：
```python
def evaluate_trading_ability(fund_code, n_quarters=4):
    """
    评估最近N个季度的交易能力
    
    Returns:
        {
            "new_buy_performance": {...},  # 新买入股票的30天表现
            "sell_performance": {...},    # 卖出股票的30天表现
            "ability_score": float,        # 能力评分（0-100）
            "ability_tag": str,           # 标签（"精准逃顶" / "频繁接盘" / "平庸交易"）
        }
    """
    # 1. 获取最近N个季度的持仓
    # 2. 对比相邻两期，识别：
    #    - 新买入：上期不在，本期在前十
    #    - 卖出：上期在前十，本期不在
    # 3. 对每笔交易，获取交易后30天股价走势
    # 4. 计算收益率：
    #    - 新买入：30天收益率 > 0 → 抄底成功
    #    - 卖出：30天收益率 < 0 → 逃顶成功
    # 5. 综合评分：
    #    - 抄底成功率 × 40% + 逃顶成功率 × 60%（逃顶更重要）
```

**可视化方案**：
```python
# 图表类型：分组柱状图
# X轴：交易类型（新买入 / 卖出）
# Y轴：30天收益率分布（箱线图或小提琴图）
# 颜色：收益 > 0（绿色） / 收益 < 0（红色）

# 辅助图表：成功率雷达图
# 维度：抄底成功率 / 逃顶成功率 / 平均盈利 / 最大亏损
```

**关键指标**：
- **抄底成功率**：新买入股票30天正收益比例
- **逃顶成功率**：卖出股票30天负收益比例
- **能力标签**：
  - `抄底 > 70% & 逃顶 > 70%` → "神级交易能力"
  - `抄底 > 60% & 逃顶 > 60%` → "优秀交易能力"
  - `抄底 < 40% & 逃顶 < 40%` → "频繁接盘侠"
  - `其他` → "平庸交易"

**边界条件**：
- 如果不满4个季度，显示"数据不足，暂不分析"
- 使用最近5年（20个季度）的持仓数据计算交易能力

### 模块4：最新持仓估值分析与风险预警
**目的**：识别高估股票，进行压力测试

#### 4.1 估值分析
**数据来源**：
- 接口：`ak.stock_zh_a_spot_em()` 获取实时PE/PB
- 行业对比：`ak.stock_zh_valuation_comparison_em()` 获取行业分位数

**实现逻辑**：
```python
def analyze_holdings_valuation(fund_code):
    """
    分析最新持仓的估值情况
    
    Returns:
        {
            "overvalued_stocks": [...],    # 高估股票列表（PE > 行业80分位）
            "fair_valued_stocks": [...],   # 估值合理股票
            "undervalued_stocks": [...],   # 低估股票
            "risk_warning": str,           # 风险预警
        }
    """
    # 1. 获取最新前十大持仓
    # 2. 对每只股票：
    #    - 获取实时PE/PB
    #    - 获取对应行业的PE分位数（需行业映射）
    # 3. 分类：
    #    - 高估：PE > 行业80分位 且 PE > 50
    #    - 低估：PE < 行业20分位 且 PE < 20
    #    - 其他：合理估值
    # 4. 生成风险预警（如果有高估股票）
```

**可视化方案**：
```python
# 图表类型：散点图（PE vs 占比）
# X轴：PE值（对数坐标）
# Y轴：占净值比例
# 颜色：高估（红色） / 合理（黄色） / 低估（绿色）
# 大小：持仓占比（气泡大小）

# 辅助：估值分布表
# 股票代码 | 股票名称 | PE | 行业PE分位 | 估值评级 | 占比
```

#### 4.2 压力测试
**目的**：模拟极端情况对净值的影响

**测试场景1：行业暴跌**
```python
def stress_test_industry(fund_code, industry_params):
    """
    行业暴跌压力测试
    
    industry_params: {
        "银行": -0.08,          # 防御性行业：下跌8%
        "医药": -0.30,          # 进攻型行业：下跌30%
        "半导体": -0.30,
        ...
    }
    
    Returns:
        {
            "industry_impact": {...},     # 各行业对净值的冲击
            "total_nav_decline": float,   # 预计净值跌幅
            "risk_level": str,            # 风险等级
        }
    """
    # 1. 获取最新前十大持仓的行业配置
    # 2. 对每只股票：
    #    - 获取行业（使用 sw_industry_loader.py）
    #    - 计算冲击 = 占比 × 行业跌幅
    # 3. 汇总冲击 = Σ冲击
```

**可视化方案**：
```python
# 图表类型：水平柱状图
# X轴：对净值的影响（绝对值）
# Y轴：重仓行业
# 颜色：正向影响（绿色） / 负向影响（红色）

# 辅助：净值敏感性仪表盘
# 显示：预计跌幅 / 风险等级（低/中/高/极高）
```

**测试场景2：全市场暴跌**
```python
def stress_test_market(fund_code, market_decline=-0.10):
    """
    全市场暴跌压力测试
    
    market_decline: 市场跌幅（默认-10%）
    
    Returns:
        {
            "expected_nav_decline": float,  # 预计净值跌幅
            "vs_market": float,             # 相对市场跌幅
            "defensive_strength": str,      # 防御强度标签
        }
    """
    # 1. 假设所有持仓股跌幅 = 市场跌幅 × Beta
    # 2. Beta估算：使用基金历史Beta（equity_metrics）
    # 3. 预计净值跌幅 = Σ(占比 × 市场跌幅 × Beta_i)
    # 4. 简化版：预计净值跌幅 = 基金Beta × 市场跌幅
```

**可视化方案**：
```python
# 图表类型：仪表盘（Gauge Chart）
# 指针：预计净值跌幅
# 区间：安全（< -5%） / 警示（-5% ~ -15%） / 危险（> -15%）
```

**关键指标**：
- **高风险行业暴露**：进攻型行业（医药/半导体/新能源）占比
- **防御强度**：现金仓位 + 防御性行业仓位
- **压力测试结果**：预计净值跌幅 / 风险等级

## 七、关键风险与缓解

### 风险1：数据量巨大
- **问题**：成立10年的基金，需要加载40个季度的数据，网络请求多
- **缓解**：
  - ✅ **限制时间范围**：最多显示最近5年（20个季度），不足5年按实际情况
  - 增加数据缓存机制（Parquet格式）
  - 使用并行请求（asyncio）

### 风险2：行业映射不准确
- **问题**：股票→行业映射可能有误，影响压力测试
- **缓解**：
  - 使用 `sw_industry_loader.py` 的三级映射（代码/名称/关键词）
  - 对于无法识别的行业，归为"其他"或跳过
  - 显示映射置信度

### 风险3：估值数据获取失败
- **问题**：`ak.stock_zh_a_spot_em()` 可能返回空数据
- **缓解**：
  - 设置超时机制（10秒）
  - 对于失败的数据，使用默认值或跳过
  - 显示数据覆盖度（如"估值数据覆盖：8/10"）

### 风险4：压力测试假设过于简单
- **问题**：假设所有股票跌幅一致，不符合实际
- **缓解**：
  - 使用历史Beta调整跌幅（如Beta=1.2的股票跌幅=市场×1.2）
  - 允许用户自定义跌幅参数
  - 显示假设条件，提醒用户结果仅供参考

# 阿里云采购清单 - 详细配置

## 一、ECS 实例配置

### 基础配置
```
地域：华东1（杭州）
实例规格：ecs.c6.xlarge
  - vCPU: 4 核
  - 内存: 8 GB
  - 网络：5 Gbps
  - 处理器：主频 2.5 GHz，睿频 3.2 GHz

实例类型：按量付费（推荐）或 包年包月
  - 按量付费：~200 元/月
  - 包年：~2,400 元/年（约 200 元/月，无折扣）
  - 预留实例（1年）：~1,200 元/年（约 100 元/月，折扣 50%）

存储：ESSD 云盘 PL0
  - 容量：100 GB
  - 性能：3000 IOPS
  - 成本：~60 元/月
```

### 镜像选择
```
镜像类型：公共镜像
操作系统：Ubuntu 22.04 LTS
版本：22.04.4 LTS (amd64)
```

### 网络带宽
```
带宽类型：按使用流量计费
峰值带宽：5 Mbps
费用预估：~50 元/月（取决于实际流量）
```

### 安全组配置
```
安全组名称：fund_analyst_security

入站规则：
  协议端口          授权对象       说明
  SSH (22)         你的IP        管理访问
  自定义 TCP (5432) 172.16.0.0/24  RDS 访问（VPC 内网）
  HTTP (80)         0.0.0.0/0    允许所有访问
  HTTPS (443)       0.0.0.0/0    允许所有访问

出站规则：
  协议端口          授权对象
  全部              全部          允许所有出站
```

### 购买确认
```
实例数量：1 台
购买时长：1 个月
自动释放：否
```

**ECS 总成本**: ~310 元/月（含存储、带宽）

## 五、购买后的初始化步骤

### 第 1 天：基础环境
```bash
# 1. 登录 ECS
ssh root@<ECS公网IP>

# 2. 更新系统
apt update && apt upgrade -y

# 3. 安装基础软件
apt install -y \
    python3-pip \
    postgresql-client \
    nginx \
    supervisor \
    redis-server \
    git

# 4. 配置系统参数
echo "* soft nofile 65535" >> /etc/security/limits.conf
echo "* hard nofile 65535" >> /etc/security/limits.conf
sysctl -w net.core.somaxconn=1024
```

### 第 2 天：部署应用
```bash
# 1. 克隆代码
git clone <你的GitHub仓库>
cd fund_analyst

# 2. 安装依赖
pip3 install -r requirements.txt

# 3. 配置环境变量
export DB_HOST=<RDS内网地址>
export DB_PORT=5432
export DB_NAME=fund_analysis
export DB_USER=fund_analyst
export DB_PASSWORD=<你的密码>

# 4. 初始化数据库
python3 scripts/init_database.py

# 5. 启动更新服务
python3 scripts/scheduler.py &
```

### 第 3 天：测试和验证
```bash
# 1. 测试数据库连接
python3 -c "from cloud_db_adapter import CloudDatabase; db = CloudDatabase(...); print('OK')"

# 2. 测试 API 调用
python3 -c "from pipeline import analyze_fund; report = analyze_fund('000001', years=1); print('OK')"

# 3. 测试 Web 服务
curl http://localhost:8501
```

## 七、成本优化建议

### 短期优化（1-3个月）
1. ✅ 使用 1 年预留实例（ECS 折扣 50%）
2. ✅ 选择按量计费带宽（根据实际使用）
3. ✅ 设置 RDS 按量转包月（如长期使用）

### 中期优化（3-6个月）
1. ⚠️ 考虑将历史数据迁移到对象存储（OSS）
2. ⚠️ 优化数据库查询索引
3. ⚠️ 使用读写分离（如并发量大）

### 长期优化（6-12个月）
1. 📊 分析实际资源使用，调整配置
2. 📊 冷热数据分离（热数据 SSD，冷数据 OSS）
3. 📊 引入 Redis 缓存层

## 八、应急预案

### ECS 故障
```
症状：服务器无法访问
排查：
  1. 检查安全组规则
  2. 检查 CPU/内存/磁盘是否满
  3. 查看系统日志
  4. 重启实例

恢复：
  - 使用快照恢复
  - 重新部署应用
```

### RDS 故障
```
症状：数据库连接失败
排查：
  1. 检查白名单设置
  2. 检查连接数是否超限
  3. 查看慢查询日志

恢复：
  - 切换到从库（如有）
  - 使用备份恢复
```

# 云服务器规格推荐方案

## 场景分析

### 业务需求
- **数据类型**: 基金净值数据、持仓数据、预计算指标
- **访问特点**: 读多写少（查询频率 >> 更新频率）
- **并发规模**: 当前 < 10 req/min，目标 > 1000 req/min
- **数据规模**:
  - 基金数量: ~15,000 只
  - 净值历史: 每只基金平均 10 年 × 250 天 = 2,500 条
  - 总净值记录: ~37,500,000 条
  - 预计算指标: ~15,000 基金 × 4 个周期 = 60,000 条
  - 持仓数据: ~15,000 基金 × 4 期/年 × 5 年 = 300,000 条
  - **预估总存储**: ~50GB

## 详细配置方案

### 推荐配置：方案 A（阿里云 ECS + RDS）

#### 1. ECS 配置

```yaml
# ECS 实例
规格: ecs.c6.xlarge (4 vCPU, 8GB 内存)
存储: 100GB ESSD
操作系统: Ubuntu 22.04 LTS
带宽: 5Mbps（按使用量计费）

# 安全组配置
入站规则:
  - 端口 22 (SSH)    : 你的IP
  - 端口 5432 (PGSQL) : VPC 内网访问
  - 端口 80 (HTTP)    : 0.0.0.0/0
  - 端口 443 (HTTPS)  : 0.0.0.0/0

出站规则: 全部允许
```

#### 2. RDS 配置

```yaml
# RDS 实例
引擎: PostgreSQL 14.x
规格: rds.pg.s2.large (2 vCPU, 4GB 内存)
存储: 100GB ESSD PL0
IOPS: 3000
备份保留: 7天
时区: UTC+8

# 白名单配置
允许访问: ECS 私有IP地址 (推荐)
       或: 你的办公IP (不推荐生产环境)

# 账号权限
数据库用户: fund_analyst (仅授予 SELECT/INSERT/UPDATE)
禁止访问: DROP/DELETE/ALTER (防止误操作)
```

#### 3. 软件环境

```bash
# ECS 需要安装的软件
Python 3.11
PostgreSQL Client
Redis 7.x (缓存层)
Nginx 1.24 (反向代理)
Supervisor (进程管理)

# 关键配置
ulimit -n 65535          # 增加文件描述符限制
sysctl -w net.core.somaxconn=1024  # TCP 队列长度
```

#### 4. 数据库调优

```sql
-- PostgreSQL 配置 (postgresql.conf)
shared_buffers = 1GB              -- 内存的 1/4
effective_cache_size = 6GB         -- 内存的 3/4
maintenance_work_mem = 256MB
checkpoint_completion_target = 0.9
wal_buffers = 16MB
default_statistics_target = 100
random_page_cost = 1.1            -- SSD 优化
effective_io_concurrency = 200
max_connections = 200
work_mem = 16MB

-- 查询优化
CREATE INDEX idx_fund_nav_code_date ON fund_nav(fund_code, date DESC);
CREATE INDEX idx_fund_metrics_code_period ON fund_metrics(fund_code, period, analysis_date DESC);
CREATE INDEX idx_fund_holdings_code_date ON fund_holdings(fund_code, report_date DESC);
```

## 成本优化建议

### 1. 使用预留实例
- **1年期**: 折扣 ~50%（节省 ~200元/月）
- **3年期**: 折扣 ~60%（节省 ~260元/月）

### 2. 按需调整规格
- 测试期: 2核4GB (~150元/月)
- 小规模: 4核8GB (~300元/月)
- 大规模: 8核16GB (~600元/月)

### 3. 混合存储
- 热数据（最近1年）: SSD 存储成本
- 冷数据（历史数据）: 对象存储（OSS/S3），成本降低 80%

## 购买步骤（以阿里云为例）

### 第一步：购买 ECS
1. 登录阿里云控制台
2. 产品 → 弹性计算 → ECS
3. 创建实例 → 选择 ecs.c6.xlarge
4. 选择镜像: Ubuntu 22.04 LTS
5. 选择存储: 100GB ESSD
6. 选择带宽: 5Mbps 按量
7. 设置密码 / SSH 密钥
8. 立即购买

### 第二步：购买 RDS
1. 产品 → 云数据库 → RDS PostgreSQL
2. 创建实例 → 选择 rds.pg.s2.large
3. 存储空间: 100GB
4. 选择 VPC（与 ECS 相同）
5. 设置密码
6. 立即购买

### 第三步：网络配置
1. 安全组配置（开放必要的端口）
2. 内网互通（ECS 和 RDS）
3. 域名解析（可选）

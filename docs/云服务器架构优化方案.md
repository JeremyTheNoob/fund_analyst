# 云服务器架构优化方案

## 方案对比

### 方案一：增量更新 + 本地数据库 ⭐⭐⭐⭐⭐（推荐）

#### 架构设计
```
AkShare API → 云数据库 → 本地缓存 → 用户请求
                ↓
           增量更新任务
        (每日/每小时)
```

#### 技术选型

**数据库选择**：
- **PostgreSQL** (推荐)
  - 支持 JSONB 存储（灵活存储不同基金类型的指标）
  - 时间序列数据索引优秀
  - 支持全文搜索（基金名称、基金经理）
  - 可以使用 TimescaleDB 扩展优化时间序列查询
  
**备选方案**：
- **MongoDB** - 灵活的文档型数据库，适合基金数据
- **ClickHouse** - 超高性能列式数据库，适合大规模时间序列

#### API 优化

**新增云数据库接口**：
```python
# cloud_db_adapter.py
from sqlalchemy import create_engine, text
from sqlalchemy.orm import sessionmaker
import pandas as pd

class CloudDatabase:
    def __init__(self, db_url):
        self.engine = create_engine(db_url)
        self.Session = sessionmaker(bind=self.engine)
    
    def get_fund_nav(self, fund_code, start_date=None, end_date=None):
        """获取基金净值数据"""
        query = text("""
            SELECT date, unit_nav, accumulated_nav, daily_return
            FROM fund_nav
            WHERE fund_code = :fund_code
            AND (:start_date IS NULL OR date >= :start_date)
            AND (:end_date IS NULL OR date <= :end_date)
            ORDER BY date
        """)
        with self.Session() as session:
            result = session.execute(query, {
                'fund_code': fund_code,
                'start_date': start_date,
                'end_date': end_date
            })
            return pd.DataFrame(result.fetchall(), columns=['date', 'unit_nav', 'accumulated_nav', 'daily_return'])
    
    def get_fund_metrics(self, fund_code, period):
        """获取预计算的指标"""
        query = text("""
            SELECT * FROM fund_metrics
            WHERE fund_code = :fund_code AND period = :period
            ORDER BY analysis_date DESC
            LIMIT 1
        """)
        with self.Session() as session:
            result = session.execute(query, {'fund_code': fund_code, 'period': period})
            row = result.fetchone()
            if row:
                return dict(row)
            return None
    
    def save_fund_metrics(self, fund_code, period, metric_type, metrics):
        """保存基金指标"""
        with self.Session() as session:
            # 删除旧数据
            session.execute(text("""
                DELETE FROM fund_metrics 
                WHERE fund_code = :fund_code AND period = :period
            """), {'fund_code': fund_code, 'period': period})
            
            # 插入新数据
            session.execute(text("""
                INSERT INTO fund_metrics (
                    fund_code, period, metric_type, analysis_date,
                    total_return, annualized_return, max_drawdown,
                    volatility, sharpe_ratio, alpha, beta, r_squared,
                    information_ratio, sortino_ratio, recovery_days,
                    monthly_win_rate, extra_metrics
                ) VALUES (
                    :fund_code, :period, :metric_type, NOW(),
                    :total_return, :annualized_return, :max_drawdown,
                    :volatility, :sharpe_ratio, :alpha, :beta, :r_squared,
                    :information_ratio, :sortino_ratio, :recovery_days,
                    :monthly_win_rate, :extra_metrics::jsonb
                )
            """), {
                'fund_code': fund_code,
                'period': period,
                'metric_type': metric_type,
                **metrics
            })
            session.commit()
```

## 方案二：完全预计算 + REST API

### 架构设计
```
用户请求 → FastAPI 后端 → 云数据库 → 预计算结果
                ↓
            定时任务
        (全量计算)
```

### 优点
- 响应速度极快（< 100ms）
- 不依赖 AkShare API
- 可以做复杂的离线分析

### 缺点
- 实时性差（T+1）
- 存储成本高
- 实施复杂度高

## 方案三：混合方案（最终推荐）

### 架构设计
```
用户请求 → Streamlit → 检查云数据库
                       ↓
                   有数据？   没有？
                     ↓         ↓
                  直接返回    API + 计算
                     ↓         ↓
                     +--------+
                     ↓
                   异步更新数据库
```

### 实施策略
1. **热数据缓存**：最近访问的 1000 只基金，数据保留在云数据库
2. **冷数据回退**：不常用的基金，使用 API 实时计算
3. **后台预热**：每天凌晨预热热门基金数据
4. **智能预测**：根据用户行为，提前计算可能访问的基金

## 风险与注意事项

1. **数据一致性**：AkShare 数据可能回溯修正，需要处理版本控制
2. **API 限制**：即使有云数据库，仍需遵守 AkShare 的使用条款
3. **成本控制**：监控 API 调用量和数据库存储成本
4. **法律合规**：注意基金数据的版权和使用限制
5. **数据备份**：定期备份数据库，防止数据丢失

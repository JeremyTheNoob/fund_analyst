## 流动性与PE分位分析实现（2026-03-26 22:15）

### 新增模块

**文件**: `data/liquidity_analysis.py`（500+行）

#### 1. 拥挤度计算功能

**核心逻辑**：
```
变现天数 = (基金规模 × 持仓占比) / (日均成交额 × 参与率)

其中：
- 基金规模：单位亿元
- 持仓占比：小数形式
- 日均成交额：过去20天平均成交额，单位亿元
- 参与率：每日成交占用上限，默认 10%（0.1）
```

**风险等级**：
- < 2天：低（优秀）
- 2-5天：中（中等）
- 5-7天：高（较高）
- > 7天：极高（易进难出）

**核心函数**：
- `fetch_stock_daily_volume()` - 获取个股过去N天的日成交量数据
- `calculate_crowding_metrics()` - 计算基金持仓拥挤度及变现天数
- `calculate_crowding_by_market_cap()` - 替代方案（基于市值）
- `fetch_top10_liquidity_metrics()` - 批量获取前十大持仓的流动性指标

#### 2. PE分位优化功能

**数据源**：AkShare `stock_a_indicator` 接口（告别随机数）

**计算逻辑**：
```
PE分位 = (加权PE / 市场平均PE) × 50（基准50分）
限制在 0-100 范围内

加权PE = Σ(个股PE × 持仓占比) / Σ(持仓占比)
```

**估值等级**：
- > 80%：估值过热
- 60%-80%：偏高
- 40%-60%：合理
- 20%-40%：偏低
- < 20%：低估

**核心函数**：
- `fetch_stock_indicator()` - 获取个股估值指标（PE/PB/市值）
- `calculate_portfolio_pe_percentile()` - 计算组合加权PE分位
- `fetch_portfolio_valuation_metrics()` - 批量获取组合估值指标

#### 3. 风险预警标签生成

**双重高压标签**：
```
变现天数 > 5天 且 PE分位 > 85%
→ 🚨 流动性与估值双重高压
```

**单一风险标签**：
- 变现天数 > 7天 → 🔴 流动性风险极高
- 变现天数 > 5天 → 🟠 流动性风险较高
- PE分位 > 80% → 🔴 估值过热
- PE分位 > 60% → 🟠 估值偏高

**双优标签**：
```
变现天数 < 2天 且 PE分位 < 40%
→ ✅ 流动性与估值双优
```

## 升级至 v12.0：指数/ETF基金效率分析引擎（2026-03-26 14:59）

### 新增模块

#### Index Efficiency Engine（`models/index_efficiency_engine.py`）

工业级过程导向分析体系，从"结果导向"升级为"过程+成本导向"

**核心升级点**：
- **不再只看"跟得像不像"**，而是分析"为什么跟不像" + "持有成本是多少"
- **四大关键维度**：
  1. **效率指标**：现金拖累 / 分红再投资效率 / 调仓冲击
  2. **交易属性**：折溢价率稳定性 / 流动性画像（Bid-Ask Spread）
  3. **指数增强型归因**：Alpha 稳定性（IR）/ 月度胜率 / 主动偏离度
  4. **QDII/商品特有维度**：汇率贡献度 / 展期收益

**核心功能**：
1. **现金拖累分析** (`analyze_cash_drag`)
   - 公式：`Drag = (1 - 仓位) * (指数收益 - 现金收益)`
   - 分时间窗口分析（最近1月/3月/全部）

2. **滚动回归仓位估算** (`estimate_fund_position`)
   - 使用 scipy.optimize 最小化残差平方和
   - 约束：w_stock + w_cash = 1，禁止做空
   - 推荐：20日窗口（平衡灵敏度和稳定性）
   - 输出：实时仓位 + SMA 5日平滑

3. **多指数风格分析** (`estimate_multi_factor_style`)
   - 同时估算仓位 + 风格偏移（如 HS300/ZZ1000 拆分）
   - 适用于全市场选股的混合基金或宽基指数增强

4. **实时现金拖累校准** (`calibrate_live_cash_drag`)
   - 动态修正现金拖累模型
   - 相比使用过时的季报仓位，归因精度提升 60%+

  - 换手率估算（用持仓变动率替代，因AkShare无换手率接口）
  - 抱团度监控（前十大重仓股的Top50大市值股票占比）

#### 3. 财务质量评分（`calculators/quality_scorer.py`）
- **功能**：基于AkShare基础财报接口合成Piotroski F-Score
- **9指标体系**：
  - 盈利能力（4分）：ROA>0, CFO>0, ΔROA>0, Accrual>0
  - 杠杆与流动性（3分）：ΔLeverage<0, ΔCurrentRatio>0, EquityIssuance=0
  - 运营效率（2分）：ΔGrossMargin>0, ΔAssetTurnover>0
- **特殊情况处理**：
  - 金融行业：跳过F-Score，标记为N/A
  - 数据缺失：新上市不满2年，变动类指标默认0.5分
  - 年报未出：自动回退到最新季报
- **API**：
  - `FScoreCalculator`: F-Score计算器类
  - `analyze_portfolio_quality()`: 分析组合财务质量

#### 4. 流动性压力测试（`models/liquidity_stress_test.py`）
- **功能**：评估债基的流动性风险和赎回压力
- **核心维度**：
  - 基金规模敏感性（规模越小，流动性风险越高）
  - 历史赎回压力（大额赎回日的回撤）
  - 持有人结构估算（机构持有人占比>90%预警）
- **API**：
  - `estimate_liquidity_risk()`: 估算流动性风险
  - `calculate_liquidity_adjusted_risk()`: 计算经流动性调整的风险指数
  - `analyze_holder_structure()`: 分析持有人结构
  - `run_liquidity_stress_test()`: 执行流动性压力测试

def translate_style_drift(style_consistency: dict) -> str:
    """翻译风格漂移结果"""
    warnings = style_consistency.get('warnings', [])

    if not warnings:
        return "风格稳定，未发现漂移现象"

    text = f"检测到{len(warnings)}项风格异常：\n"
    for i, warning in enumerate(warnings, 1):
        text += f"{i}. {warning}\n"

    return text

def translate_trading_behavior(trading_behavior: dict) -> str:
    """翻译交易行为分析结果"""
    conclusions = trading_behavior.get('conclusions', [])

    if not conclusions:
        return "交易风格数据不足，无法评价"

    text = "交易行为分析：\n"
    for i, conclusion in enumerate(conclusions, 1):
        text += f"{i}. {conclusion}\n"

    risk_level = trading_behavior.get('risk_level', 'low')
    risk_factors = trading_behavior.get('risk_factors', [])

    text += f"\n风险评级：{risk_level}\n"
    if risk_factors:
        text += f"主要风险因素：{', '.join(risk_factors)}"

    return text

def translate_portfolio_quality(portfolio_quality: dict) -> str:
    """翻译财务质量评分结果"""
    weighted_f_score = portfolio_quality.get('weighted_f_score', 0)
    high_quality_weight = portfolio_quality.get('high_quality_weight', 0)
    low_quality_weight = portfolio_quality.get('low_quality_weight', 0)
    conclusions = portfolio_quality.get('conclusions', [])

    text = f"持仓财务质量评分：{weighted_f_score}/9\n"
    text += f"- 高质量股票权重：{high_quality_weight}%\n"
    text += f"- 低质量股票权重：{low_quality_weight}%\n\n"

    for conclusion in conclusions:
        text += f"• {conclusion}\n"

    return text

## 计算与估计指标清单（2026-03-26 19:36）

### 新增文档

**文件**: `计算与估计指标清单.md`

**内容**: 系统中共有 **99 项**需要计算或估计的指标，分为17大类：

### 核心计算指标

1. **收益率计算**（5项）
   - 日收益率、累计收益、年化收益、滚动收益、月度收益
   - 方法：`pct_change()`, `cumprod()`, `resample()`

2. **Fama-French因子**（5项）
   - MKT（市场）、SMB（规模）、HML（价值）、MOM（动量）、信用利差
   - 方法：指数收益率相减、债券利差计算

3. **风险指标**（6项）
   - 波动率、下行波动率、最大回撤、回撤修复天数、VaR、CVaR
   - 方法：统计量计算、滚动窗口分析

4. **Alpha/Beta**（9项）
   - CAPM：Beta, Alpha（年化）, R²
   - FF三因子/五因子：α, β_MKT, β_SMB, β_HML, β_MOM
   - Treynor-Mazuy：择时系数、择时能力
   - 方法：OLS回归分析

5. **风格分析**（5项）
   - 滚动Beta（20日/60日）、Beta漂移、风格一致性、小盘暴露
   - 方法：滚动OLS回归、标准差分析
   - 约束：Beta限制在[0, 1]范围

6. **Brinson归因**（4项）
   - 配置贡献、选择贡献、交互效应、总超额收益
   - 方法：行业权重×收益率分解

### 债券与转债计算

7. **债券模型**（6项）
   - 久期归因：有效久期、凸性、信用利差敏感度
   - 三因子模型：α（年化）, β_短端, β_长端, β_信用
   - 方法：OLS回归

8. **转债模型**（9项）
   - Delta：直接获取或估算（`exp(-0.03 × 溢价率)`）
   - 估值：转股溢价率、纯债溢价率、债底保护、YTM
   - 综合指标：E_total（综合权益暴露）、有效久期
   - 方法：公式计算、经验公式

### 指数与持仓分析

9. **指数/ETF效率**（9项）
   - 仓位估算：基金仓位、现金拖累、调仓磨损
   - 增强型：信息比率IR、月度胜率、主动偏离度
   - 折溢价：折溢价率、折溢价稳定性
   - 方法：滚动OLS回归、统计计算

10. **持仓分析**（6项）
    - 行业权重、集中度（HHI）、前十大持仓占比
    - 基本面：PE分位、ROA、资产负债率、流动比率
    - 方法：加权平均、统计计算

### 高级分析

11. **宏观指标**（3项）
    - 收益率曲线斜率、信用利差、市场波动率
    - 方法：差值计算、滚动标准差

12. **雷达图评分**（10项）
    - Alpha雷达：超额收益、风险调整后收益、风格稳定性、择时能力、月度胜率
    - 纯债雷达：信用质量、集中度、结构合理性、久期择时
    - 方法：归一化评分（0-100分）

13. **压力测试**（6项）

    - 历史情景：2016钱荒、2022理财赎回、2024小盘流动性

14. **流动性评估**（2项）
    - 流动性风险、流动性调整后风险
    - 方法：估值/规模比值

15. **交易行为**（2项）
    - 换手率、拥挤度评分
    - 方法：持仓变动分析

16. **估值预警**（2项）
    - PE分位、估值水平（高位/低位/中位）
    - 注意：由于API限制，PE分位使用随机数模拟（10%-90%）

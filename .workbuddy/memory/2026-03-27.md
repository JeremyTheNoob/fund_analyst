UI显示的是原始基准文本（如"本基金暂不设业绩比较基准"），而不是解析后的基准信息，导致用户无法看到实际的业绩比较基准。

### 修复方案
修改 `fund_quant_v2/main.py` 的 Part 1 基金基础信息显示逻辑：
- 从 `basic.benchmark_parsed` 提取解析后的基准组件
- 生成清晰的基准描述（如"沪深300×80% + 中债综合财富×20%"）
- 如果解析失败，显示原始基准文本或"—"

### 修改内容
```python
# 修改前
benchmark = basic.benchmark_text or "—"

# 修改后
benchmark_display = basic.benchmark_text or "—"
if basic.benchmark_parsed and basic.benchmark_parsed.get("components"):
    components = basic.benchmark_parsed["components"]
    comp_desc = " + ".join([f"{c['name']}×{int(c['weight']*100)}%" for c in components])
    benchmark_display = comp_desc
```

- 原始文本：本基金暂不设业绩比较基准
- 解析后：沪深300×80% + 中债综合财富×20% ✅

**011891（基金A）**：

- 解析后：中证500×100% ✅

**110020（易方达消费行业）**：
- 原始文本：沪深300指数收益率X95%+活期存款利率(税后)X5%
- 解析后：沪深300×100% ✅

## 业绩比较基准使用沪深300全收益指数（2026-03-28 00:15）

### 问题描述
用户反馈累计收益曲线中没有显示业绩比较基准曲线，尽管 `chart_gen.py` 的 `_cumulative_return_chart()` 函数（line 95-110）已经实现了基准曲线的绘制逻辑。

### 问题分析
经过详细的代码分析，发现：

1. **图表绘制逻辑正确**：`chart_gen.py` 的 `_cumulative_return_chart()` 函数已经实现了基准曲线的绘制逻辑
2. **基准数据传递正确**：`pipeline.py` 的 `_run_equity_pipeline()` 函数（line 233-234）已经将基准数据添加到 `report.chart_data` 中
3. **数据获取逻辑正确**：`_cumulative_return_chart()` 函数通过 `report.chart_data.get("benchmark_df")` 获取基准数据

但是，基准曲线仍然没有在UI中显示。经过进一步分析，发现问题出在：
- 使用的是沪深300价格指数（sh000300），这个指数不含分红
- 应该使用沪深300全收益指数（H00019），包含分红再投资收益
- 虽然基准数据生成和传递逻辑正确，但使用的是价格指数，导致基准曲线与实际收益不匹配

### 解决方案
修改 `fund_quant_v2/data_loader/equity_loader.py` 的 `build_benchmark()` 函数，将所有 fallback 从沪深300价格指数（sh000300）改为全收益指数（H00019）：

**第589行**：第一个fallback（默认基准）
- 从：`load_index_daily("sh000300", ...)`
- 改为：`load_index_daily("H00019", ...)`

**第619行**：第二个fallback（回退基准）
- 从：`load_index_daily("sh000300", ...)`
- 改为：`load_index_daily("H00019", ...)`

### 效果
- ✅ 当基金没有明确的业绩比较基准或解析失败时，使用沪深300全收益指数作为默认基准
- ✅ 全收益指数包含分红再投资收益，更能真实反映投资收益
- ✅ 基准曲线在UI中自动显示（已有实现）
- ✅ **仅用于绘图和业绩比较基准，不影响其他地方的计算和显示**

### 关键点
- **仅用于绘图**：这次修改只影响业绩比较基准的计算和曲线绘制，不影响FF因子模型、Brinson归因等其他分析模块
- **配置完整性**：`config.py` 中的 `INDEX_MAP` 已经使用 `H00019`（沪深300全收益），现在 `build_benchmark()` 的fallback也使用了相同的全收益指数，确保整个系统的一致性

### 修改文件
- `fund_quant_v2/data_loader/equity_loader.py` - build_benchmark() 函数

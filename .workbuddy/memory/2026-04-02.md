## 006502（财通集成电路产业股票A）问题诊断

### 基金基本信息
- raw_type: 股票型, framework_id: stock, 分析模型: equity (FF多因子)
- benchmark_text: 中证500指数收益率*80%+上证国债指数收益率*20%
- benchmark_parsed: {type: stock_index, components: [{name: 中证500, code: 000905.SH, weight: 1.0}]}
  - ⚠️ 权重被归一化为100%（原始80%股票+20%国债），丢失了国债20%部分
- 成立日: 2018-11-29, 分析区间: ~7.34年

### 问题1：年化收益率 18.1% vs 用户计算的 ~10%
**诊断结果：代码逻辑本身无误，但数据解读有误导**
- `annualized_return()` 使用正确的几何年化公式 `(1+R)^(252/N)-1`
- 实际运行结果：0.1812 (18.12%)
- 用户手动计算：`(4.3574/1.0)^(1/7.34)-1 = 22.21%`
- 关键矛盾：净值首日1.0→末日4.3574 = 335.74% 累计，但 `cumulative_return` 仅为 2.145 (214.5%)
- **原因**：`cumulative_return` 基于**日收益率序列累乘** `∏(1+r_i)-1`，而非 首末日净值比。差异来自 processor 层的**去极值/清洗**操作或基金分拆/分红等导致 `ret` 序列与净值不一致
- 年化收益率 18.1% 是基于 214.5% 累计收益的正确几何年化，但**报告中展示的累计收益率 214.5% 与净值曲线的 335.74% 不一致**，这是核心矛盾

### 问题2：Alpha 15.8% 和分红补偿计算
**诊断结果：计算逻辑有缺陷**
- Alpha = 0.1583 (15.83%)，来自 FF 多因子回归截距 ×252（年化）
- 分红补偿 ≈ 18.3%：来自 `chart_interpretation.py` 中 `_interpret_cumulative_return()`
  - 006502 名称含"集成/电路"，匹配不到任何行业关键词，使用默认 2.5% 年化
  - 2.5% × 7.34年 = 18.35% → 这就是 18.3% 的来源
- **问题**：分红补偿是基于行业经验值的粗估，不是从数据算出来的；"真实Alpha 15.8%"直接用了FF回归alpha（没有减去分红），变量名 `net_alpha` 有误导性
- `chart_interpretation.py` 的计算逻辑是 `net_alpha = max(0, alpha - div_contribution)`，如果执行了这个逻辑则 15.8% - 18.3% = 0%，被 max(0, ...) 截断为 0%
- 用户说的 "Alpha应该是9.13%" 可能是用 (214.5% - 70.14%)/年数 ≈ 基于累计超额的年化计算

### 问题3：持仓穿透历史遗留
- "持仓数据暂不可用" 来自 `equity_report_writer.py:666`
- 检查 `report.holdings` 属性不存在（FundReport 没有 holdings 字段），代码用 `hasattr` 保护
- 实际上持仓数据在 `report.chart_data['holdings']` 中（有10条数据），但 deep holdings 组件期望的路径不同
- 这是历史遗留代码，应删除该占位文本

### 问题4：前十大持仓股历史变化图显示"-"
- 需要查看 equity_holdings_v2 相关组件的数据来源
- 可能原因：历史持仓数据中某些字段缺失

### 问题5：Brinson 归因三效应之和 10.81% vs Alpha 15.8%
**诊断结果：设计层面不一致，但非Bug**
- Brinson: allocation=1.36% + selection=6.09% + interaction=3.37% = 10.81%
- Alpha: 15.83% (FF回归年化)
- **两者含义不同**：Brinson 是资产配置归因（股债权重×收益差），Alpha 是多因子回归残差
- Brinson 使用默认参数 `bm_stock_weight=60%`，与实际基准 80% 中证500不匹配
- Brinson 应该基于 006502 的实际基准（80%中证500）来计算，而非默认的 60/40
- 分红补偿在 Brinson 和 Alpha 之间的对账逻辑缺失

## 纯债型基金分析修复（2026-03-30 16:20）

### 问题描述
输入基金000069（国投瑞银中高等级债券A）时，提示"暂不支持该基金的深度分析"。

### 根本原因
1. **bond_engine.py 崩溃**：`_estimate_duration_from_holdings()` 函数中，债券权重和异常（113.96 → 33.35 → 9.06），导致assert失败
2. **权重格式问题**：API返回的债券持仓数据中`占净值比例`是百分比格式（如8.38%），但代码直接当作小数使用
3. **持仓数据不完整**：AkShare API返回的债券持仓只是前N大重仓（50-57只），不是完整持仓，导致权重和远小于1.0

### 修复方案

#### 修复1：bond_engine.py - 权重格式兼容
**位置**：`_estimate_duration_from_holdings()` 函数

**问题**：
- API返回的`占净值比例`是百分比格式（8.38表示8.38%）
- 但代码中需要归一化权重（总和应为1.0）

**修复**：
```python
# 兼容百分比格式和小数格式：如果 ratio > 1.5，则为百分比，需除以100
if ratio > 1.5:
    ratio = ratio / 100.0
```

#### 修复2：bond_engine.py - 持仓数据不完整兼容
**问题**：
- AkShare API返回的债券持仓只是前N大重仓，不是完整持仓
- 导致权重和远小于1.0（如34.51%而不是100%）
- 强制要求权重和=1.0会导致assert失败

**修复**：
```python
# 如果权重和显著小于1.0（如<0.5），说明数据不完整，使用默认久期
if total_w < 0.5:
    logger.warning(f"[_estimate_duration_from_holdings] 权重和过小: {total_w:.4f}，持仓数据不完整，使用默认久期3.5年")
    default_dur = 3.5
    return default_dur, default_dur ** 2 / 100

# 如果权重和大于1.5，可能是百分比格式，归一化
if total_w > 1.5:
    logger.warning(f"[_estimate_duration_from_holdings] 权重和过大: {total_w:.4f}，疑似百分比格式，归一化")
    weights = [w / total_w for w in weights]
    total_w = 1.0
```

#### 修复3：bond_loader.py - 只使用最新季度数据
**问题**：
- API返回一年所有季度的数据（216只债券）
- 代码直接将所有数据存储到`bond_details`，导致重复和权重异常

**修复**：
```python
# 只保留最新季度的数据
if "季度" in df_bond.columns:
    # 找到最新的季度
    latest_quarter = df_bond["季度"].iloc[0]  # API 返回的第一个季度通常是最新的
    df_bond_latest = df_bond[df_bond["季度"] == latest_quarter].copy()
    logger.info(f"[load_bond_holdings] {symbol} 使用最新季度: {latest_quarter}, 债券数量: {len(df_bond_latest)}")
else:
    df_bond_latest = df_bond

r["bond_details"] = df_bond_latest.to_dict("records")
```

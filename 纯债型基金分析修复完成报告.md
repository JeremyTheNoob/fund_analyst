## 问题描述

## 根本原因分析

### 1. bond_engine.py 崩溃（直接原因）

**错误信息**：
```
AssertionError: 债券权重和异常: 113.96，应接近 1.0
```

**触发位置**：`engine/bond_engine.py` 第248行

**原因链**：
1. AkShare API 返回的债券持仓数据中，`占净值比例` 是**百分比格式**（如 8.38 表示 8.38%）
2. `bond_engine.py` 中的 `_estimate_duration_from_holdings()` 函数直接将这个值当作小数使用
3. 权重和计算为 113.96（应该是 1.1396），触发 assertion 失败
4. 异常被 pipeline 的 `except Exception` 捕获，返回 `_error_report()`
5. `FundReport.fund_type` 被设置为 "unknown"
6. `main.py` 中所有类型判断都失败，显示"暂不支持该基金类型的深度分析"

### 2. 持仓数据不完整（根本原因）

**API 限制**：
- AkShare 的 `_ak_fund_holdings_bond()` 接口返回的是**前N大重仓债券**，不是完整持仓
- 基金000069返回 216 只债券（4个季度），但最新季度只有 53 只
- 最新季度的债券持仓占比总和只有 34.51%（纯债基金应该在 80-90%）

**数据结构**：
```
不同季度:
  2024年1季度债券投资明细    57只（28.04%）
  2024年2季度债券投资明细    50只（21.80%）
  2024年3季度债券投资明细    56只（29.61%）
  2024年4季度债券投资明细    53只（34.51%）  ← 最新季度
```

### 3. 季度数据重复（次要原因）

**bond_loader.py 问题**：
- 第61行直接将 `df_bond.to_dict("records")` 存储到 `bond_details`
- 没有区分季度，导致 216 只债券（4个季度）全部进入久期估算
- 权重和进一步放大

## 修复方案

### 修复 1：权重格式兼容（bond_engine.py）

**位置**：`engine/bond_engine.py` 第228-236行

**修复逻辑**：
```python
for bond in bond_details:
    name  = str(bond.get("债券名称", "") or "")
    ratio = float(bond.get("占净值比例", 0) or 0)
    
    # 兼容百分比格式和小数格式：如果 ratio > 1.5，则为百分比，需除以100
    if ratio > 1.5:
        ratio = ratio / 100.0
        
    if ratio <= 0:
        continue

    dur = _infer_bond_duration(name)
    durations.append(dur)
    weights.append(ratio)
```

**效果**：
- 百分比格式（8.38）自动转换为小数（0.0838）
- 小数格式（0.0838）保持不变
- 阈值 1.5 用于自动识别格式

### 修复 2：持仓数据不完整兼容（bond_engine.py）

**位置**：`engine/bond_engine.py` 第242-258行

**修复逻辑**：
```python
total_w = sum(weights)
if total_w <= 0:
    default_dur = 3.5
    return default_dur, default_dur ** 2 / 100

# 兼容API限制：持仓数据可能只是前N大重仓（非完整持仓）
# 如果权重和显著小于1.0（如<0.5），说明数据不完整，使用默认久期
if total_w < 0.5:
    logger.warning(f"[_estimate_duration_from_holdings] 权重和过小: {total_w:.4f}，持仓数据不完整，使用默认久期3.5年")
    default_dur = 3.5
    return default_dur, default_dur ** 2 / 100

# 如果权重和大于1.5，可能是百分比格式，归一化
if total_w > 1.5:
    logger.warning(f"[_estimate_duration_from_holdings] 权重和过大: {total_w:.4f}，疑似百分比格式，归一化")
    weights = [w / total_w for w in weights]
    total_w = 1.0

# 加权平均久期（使用归一化权重）
wav_duration = sum(d * w / total_w for d, w in zip(durations, weights))
```

**效果**：
- 权重和 < 0.5：使用默认久期 3.5 年（数据不完整）
- 权重和 > 1.5：归一化后使用（百分比格式）
- 权重和在 0.5~1.5 范围：正常使用

### 修复 3：只使用最新季度数据（bond_loader.py）

**位置**：`data_loader/bond_loader.py` 第57-73行

**修复逻辑**：
```python
# --- 债券持仓明细 ---
for year in ["2024", "2023", "2022"]:
    df_bond = _ak_fund_holdings_bond(symbol, year)
    if df_bond is not None and not df_bond.empty and "占净值比例" in df_bond.columns:
        # 只保留最新季度的数据（"季度"列值最大的）
        if "季度" in df_bond.columns:
            # 找到最新的季度
            latest_quarter = df_bond["季度"].iloc[0]  # API 返回的第一个季度通常是最新的
            df_bond_latest = df_bond[df_bond["季度"] == latest_quarter].copy()
            logger.info(f"[load_bond_holdings] {symbol} 使用最新季度: {latest_quarter}, 债券数量: {len(df_bond_latest)}")
        else:
            df_bond_latest = df_bond
        
        r["bond_details"] = df_bond_latest.to_dict("records")

        # 识别可转债持仓占比
        cb_mask = df_bond_latest["债券名称"].str.contains("可转债|转债|可交换", na=False) \
            if "债券名称" in df_bond_latest.columns else pd.Series(False, index=df_bond_latest.index)
        cb_total = df_bond_latest.loc[cb_mask, "占净值比例"].sum() if cb_mask.any() else 0.0
        r["cb_ratio"] = min(cb_total / 100.0, 1.0)

        # 若资产配置未能获取债券仓位，从持仓汇总
        if r["bond_ratio"] == 0.0:
            total = df_bond_latest["占净值比例"].sum()
            r["bond_ratio"] = min(total / 100.0, 1.0)
        break
```

**效果**：
- 只使用最新季度的 53 只债券，而不是 216 只（4个季度）
- 债券数量从 216 降到了 53
- 权重和从 33.35 降到了 9.06（百分比格式），归一化后为 1.0

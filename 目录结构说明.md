# 基金穿透式分析 - 目录结构说明

## 📂 根目录文件夹说明

### 🚀 生产目录
```
fund_quant_v2/          # 主工程目录（新架构）
data/                  # 数据目录（索引缓存、数据文件）
```

#### fund_quant_v2/ - 主工程目录
**用途**: 新一代基金分析系统的核心代码库  
**内容**: 完整的数据流水线 + Streamlit应用  
**结构**:
```
fund_quant_v2/
├── data_loader/       # 数据加载模块
│   ├── index_cache_config.py    # 全收益指数配置
│   ├── index_cache_manager.py   # 缓存管理器
│   ├── index_updater.py        # 更新机制
│   ├── index_integration.py     # 系统集成
│   ├── equity_loader.py        # 权益类数据加载
│   └── ...                    # 其他加载器
├── engine/            # 计算引擎
│   ├── calculator.py           # 指标计算器
│   ├── alpha_calculator.py      # Alpha计算
│   └── ...                    # 其他计算模块
├── models/            # 数据模型
│   ├── schema.py              # 数据结构定义
│   ├── fund_basic.py          # 基金基本信息
│   └── ...                    # 其他模型
├── processor/         # 数据处理
│   ├── data_cleaner.py        # 数据清洗
│   ├── data_aligner.py        # 数据对齐
│   └── ...                    # 其他处理模块
├── reporter/          # 报告生成
│   ├── chart_gen.py           # 图表生成
│   ├── report_gen.py          # 报告生成
│   └── ...                    # 其他报告模块
├── services/          # 服务模块
│   ├── chart_interpretation.py # 图表解读引擎
│   ├── advice_engine.py       # 投资建议引擎
│   └── ...                    # 其他服务
├── tests/             # 单元测试
│   ├── smoke_test.py          # 烟雾测试
│   ├── test_common_metrics.py  # 公共指标测试
│   └── ...                    # 其他测试
├── config.py          # 主配置文件
├── main.py           # Streamlit应用入口
├── pipeline.py       # 数据流水线
└── requirements.txt  # Python依赖包
```

#### data/ - 数据目录
**用途**: 存储所有数据文件和缓存  
**内容**:
```
data/
├── index_cache/              # 全收益指数缓存（新系统）
│   ├── price_indices/        # 价格指数缓存 (*.parquet)
│   ├── total_return_indices/ # 全收益指数缓存 (*.parquet)
│   └── index_metadata.json   # 指数元数据
└── (其他数据文件)           # 按需创建
```

### 📦 归档目录
```
_archive_final/        # 历史版本归档
```

#### _archive_final/ - 历史版本归档
**用途**: 保存旧版本的所有文件，用于历史记录和参考  
**内容**: 125个Python文件，54个Markdown文档，5个备份文件  
**说明**: 本地保留，不同步到GitHub

### 🔧 配置目录
```
.git/                 # Git版本控制
.github/              # GitHub配置
.streamlit/           # Streamlit配置
.workbuddy/          # WorkBuddy工作空间
```

#### .git/ - Git版本控制
**用途**: Git版本控制系统  
**内容**: Git仓库的历史记录、分支、配置等  
**说明**: 不要手动修改此目录

#### .github/ - GitHub配置
**用途**: GitHub Actions自动化工作流配置  
**内容**:
```
.github/
├── workflows/              # 工作流定义
│   ├── deploy.yml          # 自动部署配置
│   └── test.yml           # 自动测试配置
└── (其他GitHub配置)
```

#### .streamlit/ - Streamlit配置
**用途**: Streamlit应用运行配置  
**内容**: 应用设置、密钥等（通常包含secrets.toml）

#### .workbuddy/ - WorkBuddy工作空间
**用途**: AI助手的工作目录，存储对话记忆和配置  
**内容**:
```
.workbuddy/
├── memory/                # 对话记忆
│   ├── MEMORY.md          # 长期记忆（精简版）
│   ├── 2026-03-26.md     # 每日记录
│   ├── 2026-03-27.md
│   ├── 2026-03-28.md
│   └── 2026-03-29.md
└── (其他配置文件)
```

### 💾 缓存目录
```
.cache/               # Python缓存
.pytest_cache/        # Pytest测试缓存
__pycache__/         # Python字节码缓存
```

#### .cache/ - Python缓存
**用途**: Python包管理器和工具的缓存  
**说明**: 可以安全删除，重新安装时会自动生成

#### .pytest_cache/ - Pytest测试缓存
**用途**: Pytest测试框架的缓存  
**说明**: 测试运行后自动生成，加速后续测试

#### __pycache__/ - Python字节码缓存
**用途**: Python编译后的字节码缓存  
**说明**: 加速Python代码运行，但应在Git中忽略

### 🎯 重要提示

1. **主要工作目录**: `fund_quant_v2/` - 所有开发都在这里进行
2. **数据缓存**: `data/index_cache/` - 自动管理，不需要手动维护
3. **历史归档**: `_archive_final/` - 只读参考，不要修改
4. **工作空间**: `.workbuddy/` - AI助手使用，不要手动修改
5. **缓存清理**: `.cache/`, `.pytest_cache/`, `__pycache__/` 可以定期清理

### 🚀 推荐工作流程

1. **开发新功能**: 在 `fund_quant_v2/` 中进行
2. **运行应用**: `streamlit run fund_quant_v2/main.py`
3. **运行测试**: 在 `fund_quant_v2/tests/` 中执行测试
4. **查看数据**: 从 `data/` 读取必要的配置和数据
5. **参考历史**: 如需查看旧代码，查阅 `_archive_final/`

# UI修复计划 - 根据用户反馈

## 问题列表

### 1. 五维评分缺少解读
**问题描述**: 目前五维评分只有一个分数,没有关于分数的内涵和解读
**影响**: 读者无法理解分数代表什么意思
**修复方案**:
- 在 `render_radar_scores()` 中添加总分解读
  - 90+: "优秀"
  - 80-89: "良好"
  - 70-79: "中等"
  - 60-69: "一般"
  - <60: "较差"
- 添加各维度分数的简要解读说明

### 2. 业绩指标缺少解读
**问题描述**: 表格里的年化波动、夏普比率、最大回撤都不容易理解
**影响**: 非专业读者无法理解这些指标
**修复方案**:
- 在 `render_performance_metrics()` 中添加指标解读
  - 年化波动: "衡量收益波动程度,数值越低越稳定"
  - 夏普比率: "每承担1单位风险获得的超额收益,>1为优秀"
  - 最大回撤: "历史最大亏损幅度,数值越小风控越好"

### 3.1 权益类模型分析问题
**问题描述**:
- 只有FF因子模型,解读单调
- 缺少超额收益拆解
- 指标(alpha, alpha显著性, R²)缺少内涵解读
**修复方案**:
- 在 `render_ff_results()` 中添加指标解读
- 添加"超额收益拆解"板块(从 `performance_decomposition` 读取)
- 补充Alpha解读: "剔除市场因素后,基金经理主动管理创造的年化收益"
- 补充R²解读: "模型解释度,越高说明基金风格越符合经典因子模型"

### 3.2 债券模型问题
**问题描述**: 久期归因和债券三因子模型存在计算错误
**影响**: 债券分析结果不准确
**修复方案**:
- 检查 `bond_model.py` 中的久期计算逻辑
- 检查 `run_bond_three_factors()` 的回归计算
- 添加详细的错误日志以便调试

### 3.3 债券风险模型和持仓计算错误
**问题描述**:
- 极端冲击下预估回撤为0
- 纯债基金持仓只有60%
**影响**: 风险评估严重失真
**修复方案**:
- 检查 `bond_stress_test()` 的回撤计算公式
- 检查 `analyze_bond_structure()` 的持仓比例计算
- 修正纯债基金的默认持仓比例(应该是85%,不是60%)

### 3.4 行业内Alpha/跟踪误差/信息比率缺失
**问题描述**: 行业内Alpha、跟踪误差、信息比率等的计算、展示与解读缺失
**影响**: 行业型基金无法进行精准分析
**修复方案**:
- 确保 `run_sector_model()` 的结果正确传递到UI
- 在 `render_model_results()` 中添加行业模型结果展示
- 添加信息比率解读: "每承担1%跟踪误差换回的超额收益"

### 4. 免责声明位置错误
**问题描述**: 免责声明出现在"收益曲线板块的上方",应该在报告最下方
**影响**: 违反常规UI设计原则
**修复方案**:
- 从 `render_analysis_report()` 中移除 `render_disclaimer()`
- 在 `main.py` 的所有分析结束后(持仓饼图之后)添加免责声明

### 5. 收益曲线缺少基准对比
**问题描述**: 权益类和债券类收益曲线均缺少与业绩基准的对比
**影响**: 无法直观看到基金是否跑赢基准
**修复方案**:
- 修改 `main.py` 第125行,传入 `benchmark_df`
- 在 `plot_cumulative_return()` 中添加基准曲线(如果提供)
- 添加图例说明:蓝色=基金,红色=基准

### 6. 前10大持仓缺少风险预警
**问题描述**: 前10大持仓缺少风险预警
**影响**: 无法识别持仓个股的估值风险
**修复方案**:
- 在持仓饼图下方添加估值预警卡片
- 使用 `fetch_stock_valuation_alert()` 获取前10大持仓的PE历史分位
- 对于PE>80分位的个股显示警告

### 7. 缺少大白话解读
**问题描述**: 缺少深入浅出的"大白话解读",应该放在报告上方方便读者阅读
**影响**: 非专业读者难以理解专业术语
**修复方案**:
- 在 `render_analysis_report()` 最开始添加"一句话点评"板块
- 从 `translate_results()` 或模型结果中获取大白话解读
- 风格类似: "这只基金像是个'稳健派',擅长在熊市控制回撤,但牛市可能跑不赢大盘"

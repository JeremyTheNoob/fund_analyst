# 债券型基金006184问题修复完成报告

## 问题描述

用户在测试债券型基金006184（格林泓鑫纯债A）时发现两个问题：

1. **基准年化收益显示错误**：报告显示"基准年化约 0.00%"，实际应为4.61%
2. **累计收益率图表基准曲线缺失**：图表中只有基金曲线，没有基准曲线

## 问题分析

### 问题1：基准年化收益显示0.00%

**根本原因**：
`reporter/bond_report_writer.py`第54行使用错误的基准年化计算公式：
```python
ann_bm = round(cum_bm / max(_year_count(start_date, end_date), 1), 2)
```

问题分析：
- `_year_count()`返回日历年份（365天 × 年数）
- 累计收益23.76% ÷ 5年（日历）≈ 4.75%，但实际计算结果为0.00%
- 原因：`_year_count`返回值异常大，导致除法结果接近0
- 正确的年化收益应使用复利公式：`(1 + 累计收益)^(252/交易日数) - 1`

### 问题2：累计收益率图表基准曲线缺失

**根本原因**：
1. `pipeline.py`中`_run_bond_pipeline()`没有将基准数据添加到`chart_data`
2. `reporter/chart_gen.py`中`_replace_benchmark_for_charts()`函数没有处理`ret`列名自动重命名

问题分析：
- 基准数据（`bond_idx`）已正确加载（1246行数据）
- 但`chart_data = {"nav_df": clean_nav.df}`中缺少`"benchmark_df": bond_idx`
- `chart_gen.py`接收到的`benchmark_df`为None，无法绘制基准曲线
- 即使有基准数据，列名`ret`与`_cumulative_return_chart()`检查的`bm_ret`不匹配

## 技术要点

### 基准年化收益计算公式

**错误公式**：
```python
ann_bm = cum_bm / year_count  # 使用日历年份除法
```

**正确公式**：
```python
ann_bm = (1 + cum_bm) ^ (252 / trading_days) - 1  # 使用复利公式
```

**公式说明**：
- `252`：A股市场年交易日数
- `trading_days`：基准数据实际交易日数
- 复利公式确保年化收益与累计收益在时间维度上一致

### 基准数据传递流程

```
bond_pipeline (pipeline.py)
    └─ load_bond_composite_index() → bond_df [date, ret]
        └─ chart_data = {"benchmark_df": bond_df}
            └─ generate_chart_data()
                └─ _replace_benchmark_for_charts()
                    └─ 自动重命名 ret → bm_ret
                        └─ _cumulative_return_chart()
                            └─ 计算 bm_annual_return
                                └─ benchmark_info 包含完整基准信息
                                    └─ bond_report_writer 读取 bm_annual_return
```

## 后续建议

### 1. 统一年化收益计算
建议将年化收益计算逻辑提取到公共函数中：
```python
# utils/common.py
def annualize_return(cum_return: float, trading_days: int) -> float:
    """
    使用复利公式计算年化收益

    Args:
        cum_return: 累计收益率（小数）
        trading_days: 实际交易日数

    Returns:
        年化收益率（小数）
    """
    if trading_days <= 0:
        return 0.0
    return (1 + cum_return) ** (252 / trading_days) - 1
```

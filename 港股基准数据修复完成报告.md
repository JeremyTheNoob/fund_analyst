# 基金 014416 港股基准数据修复完成报告

## 问题描述

用户在测试基金 014416（泰康研究精选股票发起A）时发现：
1. 累计收益率的业绩基准曲线没有显示
2. 超额收益曲线只有一条横线（0）
3. 水下回撤的业绩比较基准曲线也没有显示

## 问题分析

### 基金基本信息
- **基金代码**：014416
- **基金名称**：泰康研究精选股票发起A
- **基金类型**：股票型（equity）
- **业绩基准**：恒生指数 (HSI.HI)

### 问题根因

基金 014416 的业绩基准是**恒生指数**，这是一个港股指数。港股基准数据加载失败导致三个问题：

#### 错误 1：港股指数代码格式错误
- 原始代码格式：`hk:HSI.HI`
- Pipeline 提取后得到：`HSI.HI`（保留 `.HI` 后缀）
- AkShare 新浪接口格式：`HSI`（不带后缀）
- 结果：接口无法识别 `HSI.HI`，返回错误数据或空数据

#### 错误 2：接口参数未传递
- `base_api.py` 的 `_ak_hk_index_daily()` 函数没有正确传递 `symbol` 参数
- 错误代码：`lambda: ak.stock_hk_index_daily_sina()`
- 正确代码：`lambda: ak.stock_hk_index_daily_sina(symbol=symbol)`
- 结果：接口返回的不是恒生指数数据

#### 错误 3：数据加载失败的连锁反应
1. `build_benchmark()` 返回的 `benchmark.df` 为空
2. Pipeline 的 `has_benchmark` 条件判断失败（第 272-276 行）
3. `benchmark_df` 未添加到 `chart_data`（第 288-289 行）
4. 图表生成时没有基准数据（`reporter/chart_gen.py` 第 98-109 行）
5. 结果：累计收益曲线、水下回撤图、超额收益曲线都没有基准数据

## 修复方案

### 修复 1：港股指数代码格式转换

**文件**：`data_loader/equity_loader.py`（第 651-656 行）

**修改内容**：
```python
elif code is not None and code.startswith("hk:"):
    # 港股指数不支持全收益计算，使用价格指数
    # 转换格式: HSI.HI → HSI（新浪格式）
    hk_sym = code[3:]
    if '.' in hk_sym:
        hk_sym = hk_sym.split('.')[0]
    df_part = load_hk_index_daily(hk_sym, start, end).rename(columns={"ret": "part_ret"})
    logger.info(f"[build_benchmark] 使用港股指数: {hk_sym}")
```

**修复说明**：
- 从 `hk:HSI.HI` 提取 `HSI.HI`
- 检查是否包含 `.`，如果包含则分割并取第一部分
- 最终得到 `HSI`，符合 AkShare 新浪接口格式

### 修复 2：接口参数传递

**文件**：`data_loader/base_api.py`（第 401-413 行）

**修改内容**：
```python
def _ak_hk_index_daily(symbol: str, *args, **kwargs) -> Optional[pd.DataFrame]:
    """
    港股指数日线行情（新浪）

    Args:
        symbol: 指数代码（如 "HSI"）

    Returns:
        指数行情 DataFrame
    """
    return safe_api_call(
        lambda: ak.stock_hk_index_daily_sina(symbol=symbol)
    )
```

**修复说明**：
- 更新文档说明，明确参数格式为 "HSI"
- 正确传递 `symbol` 参数给 AkShare 接口
- 使用 `safe_api_call` 包装，确保超时和错误处理
